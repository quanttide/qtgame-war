# PRD: 指挥官心态面板

## 动机

当前原型覆盖了完整的决策链路：情报→判断→命令→执行→反馈。但它描述的是一个**没有感情的决策机器**——指挥官收到伤亡报告后面无表情，控制率掉了继续下一回合。

设计文档的核心动机是：

> "明知道历史走向，却依然感到步步惊心、决策艰难"

当前的实现只做到了"明知道历史走向"（回合12的预设战场），没做到"步步惊心"和"决策艰难"。指挥官心态面板就是填补这个缺口的功能。

## 设计目标

1. 让玩家感受到指挥官是一个"人"，不是光标
2. 每个决策不仅在战场上产生后果，也在指挥官心理上留下痕迹
3. 制造"不得不做出选择"的压迫感，而不是"选最优解"的理性计算

## 面板内容

### 核心状态（数值驱动）

| 指标 | 范围 | 说明 |
|------|------|------|
| 精神压力 | 0-100 | 战况越吃紧、伤亡越大，压力上升。超过阈值触发负面效果 |
| 疲劳度 | 0-100 | 连续作战、睡眠不足。影响决策精度 |
| 信心 | 0-100 | 连续失败下降，成功回升。影响部队士气 |
| 牵挂 | 文本 | 指挥官的家庭、私人事务。随着剧情推进触发 |

精神压力和信心是战场驱动的——每回合根据战报自动计算。疲劳度是时间驱动的——第5/10/15回合强制上升。牵挂是叙事驱动的——特定回合触发特定事件。

### 私人事务系统

每 3-4 回合插入一条"私人事务"提示，形式为指挥官收到的信件/电报：

```
回合 4 —— 来自后方的家信
"父亲：家中一切安好。你母亲问你是否需要
冬天的衣物。前线冷吗？—— 小妹"

[看完收起] → 压力 -5，牵挂 +1
```

私人事务的作用：
- 制造"两个世界"的撕裂感——战场上的你 vs 生活中的你
- 提醒玩家指挥官是有家庭的人，不是战略游戏里的一个头像
- 自然的压力调节机制——不是所有压力都来自战斗

### 决策的心理代价

每次执行命令后，除战场结果外，额外显示心理影响：

```
结果：正面推进 - 成功
战场效果：我军占领(3,3)(3,4)，敌军伤亡12%，我方伤亡8%
心理影响：伤亡报告让你整夜没睡。压力 +15，信心 +5
```

A 命令（正面推进/人海战术）比 B 命令（侧翼包抄/巧打）造成更多心理压力，因为正面推进伤亡更直接。
C 命令（防守反击）如果失败，对信心的打击是 A 或 B 失败的两倍——因为"等着挨打还打输了"。

## UI 方案

### 位置

指挥官心态面板放在**左面板顶部**，或者作为**HUD 的一部分叠加在地图左上角**。

建议采用 HUD 叠加方式，因为：
- 心态是一个"始终在线"的信息，不应该藏在 Tab 后面
- 做成 HUD 后玩家随时能看到压力值在涨，制造持续的压迫感
- 私人事务可以用弹窗或侧边通知呈现

### 视觉风格

```
┌──────────────────┐
│ 指挥官状态        │
│ 精神压力 ████░░ 62 │   ← 色条，压力 > 70 变红
│ 疲劳度   ██░░░░ 28 │
│ 信心     █████░ 85 │   ← 信心 < 30 闪烁警告
│ 牵挂: 家信未读 ✉  │   ← 有未读私人事务时显示信封
└──────────────────┘
```

### 叙事窗口（私人事务）

```
┌─────────────────────────────────────┐
│ 📨 来自后方的信                     │
│                                     │
│ "指挥官，你女儿学会了写你的名字。   │
│  她问爸爸什么时候回来。"            │
│                                     │
│         [收起, 压力-5]              │
└─────────────────────────────────────┘
```

## 与现有系统的交互

| 系统 | 交互方式 |
|------|---------|
| 命令系统 | 每条命令执行后追加心理影响描述 |
| 情报系统 | 模糊情报/干扰信息增加精神压力，"信息战"本身是压力源 |
| 地图变化 | 己方单位损失→压力上升；占领区域→信心上升 |
| 回合推进 | 每 3-4 回合触发私人事务；每 5 回合疲劳度+15 |

## 数据模型

```js
Game.State.commander = {
    stress: 45,        // 精神压力 0-100
    fatigue: 20,       // 疲劳度 0-100
    confidence: 80,    // 信心 0-100
    concerns: [],      // 私人事务列表 [{ turn, type, content, read }]
    lettersUnread: 0,  // 未读私信数
};
```

## 成功标准

1. 玩家是否会在做决策时犹豫——不是犹豫"哪个选项胜率高"，而是犹豫"这个选择我承受得起吗"（心理成本）
2. 私人事务是否让玩家短暂跳出"战棋模式"回到"人的模式"
3. 压力值是否自然波动，而不是线性上升或一直低位

## 不作范围

- 不做指挥官性格定制（对话选项、背景故事选择）
- 不做心理健康"游戏结束"机制（压力爆表不会 game over，只影响决策）
- 不做多指挥官系统
