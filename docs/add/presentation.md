# 表现层设计

表现层负责两件事：将状态渲染为像素和文本，将用户输入转化为事件。

---

## 渲染器

定义 `IRenderer` 接口暴露 `render(state)`，具体实现可为 Canvas 2D、SVG、WebGL、Flutter CustomPaint 等。

渲染流程：
1. 绘制六角格网格（底色 + 地形纹理 + 描边）
2. 绘制地形图标（文字或矢量符号）
3. 绘制单位（兵种符号、血条、行动标记、编制状态）
4. 绘制战争迷雾（未揭示敌军显示"?"，低可信度情报用褪色/闪烁）
5. 绘制高亮覆盖层（移动范围、攻击范围、选中）
6. 绘制战役结束覆盖层

为提高性能，渲染器可只重绘脏区域，或在状态未变化时跳过绘制。

---

## 叙事生成器（战役模式独有）

位于表现层顶部，作为视图模型转换器：
- 读取当前状态和最近事件
- 调用模板引擎 / 文本编织器
- 生成情境描述、选项文案和后果反馈文本

业务逻辑不拼接字符串，叙事的"翻译"全部在这里完成。

---

## 输入适配器

`IInputAdapter` 接口，接收原始坐标并转化为游戏事件：
- 屏幕坐标 → `pixel_to_hex()` → 命中格子
- 结合当前选中状态和高亮范围 → 决定动作类型 → 生成事件
- 右键 / 长按取消选中
- 多指手势（缩放/平移）独立处理，不干扰游戏逻辑

---

## UI 组件化

信息面板拆分为观察者组件，每个只依赖必要的状态切片：

| 组件 | 依赖的状态 |
|------|-----------|
| 态势面板 | 地形、单位位置、高亮 |
| 决策面板（战役模式） | 当前意图、可用选项、后果预览 |
| 单位卡片 | 选中单位的属性、状态 |
| 日志面板 | 操作日志时间线 |
| 指挥官状态 HUD（战役模式） | 精神压力、疲劳、信心 |

组件不包含游戏逻辑，点击按钮时直接向调度层发送对应事件。所有动态样式（高亮、禁用、文字颜色）基于状态计算，不存储本地状态。
