# 可扩展性设计

## 多尺度兼容

通过配置数据支持从排级到军级的不同战役尺度，引擎核心不修改：

- **地图描述文件**：定义格子代表的实际距离（30 米 → 3 公里），地形定义相应调整
- **单位模板**：定义单位类型（步兵排、炮兵连、团、师）的基础属性
- **规则参数表**：命中率公式、移动消耗、视野距离等全部外部化，方便调整

---

## 剧本系统

每个剧本（Scenario）独立配置，引擎根据剧本初始化状态：

| 配置项 | 说明 |
|--------|------|
| 地图文件 | 地形二维数组 + 坐标修正 |
| 初始单位部署 | 位置、属性、阵营 |
| 全局战役参数 | 时间限制、援军触发回合、特殊规则 |
| 胜利条件函数 | 可替换的胜利判定逻辑 |
| 指挥官选项集 | 仅战役模式，定义可用的意图和决策树 |

引擎不限定剧本，只负责根据剧本初始化 GameState。

## AI 策略化

AI 不硬编码为 if-else 链。通过 `ICommander` 接口实现策略模式：

```
interface ICommander {
    generate_actions(state) -> List<Action>
}
```

- **防御性 AI**：固守核心，被动反击
- **进攻性 AI**：主动向玩家薄弱点移动并攻击
- **事件驱动 AI**：检测到援军到达后切换策略
- 可在战役不同阶段动态替换策略

---

## 网络对战扩展

状态是纯数据且事件驱动，引入网络层只需：

1. 终端捕获本地事件，发送至服务器
2. 服务器持有权威 GameState，应用事件并同步给双方
3. 延迟隐藏通过简单的客户端预测 + 状态回滚实现
4. 事件溯源序列天然适合作为网络同步协议
