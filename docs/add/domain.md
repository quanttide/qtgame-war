# 领域模型

领域模型是现实世界概念在代码里的映射。一个概念一个文件，不按代码模式切分。

## 全景

```
兵种模板 ──→ 兵种实例 ──→ 战场（地形 + 坐标）
                    ↘          ↙
                     战斗结算
                        │
                  战役状态容器 ←── 引擎外观
                        ↑
                   战役配置（JSON）
```

实线是创建/引用，虚线是数据流。

---

## 兵种模板（UnitType）

定义兵种"是什么"。开战前定死，全程不改。

包含：名称、最大血量、攻防值、移动力、射程、特殊能力标记。

模板在 JSON 里定义，开局时一次加载。多个单位可以共用同一模板（比如两队"轻步兵"指向同一个模板对象）。

框架内置几个通用模板（轻步兵、重步兵、炮兵、骑兵、突击步兵），战役可以自己定义更多。

---

## 兵种实例（Unit）

代表战场上"这一个"单位。每局创建，每回合变化。

包含：所属阵营、引用哪个模板、当前坐标、当前血量、是否行动过、是否存活。

操作：移动（改坐标）、受伤（扣血，归零则阵亡）、标记已行动、标记暴露。

注意：实例上的"最大血量""攻击力"这些属性不直接存储，而是从模板读取。模板改一处，所有实例自动生效。

---

## 战场（Battlefield）

做了两件不同的事：

**坐标系统**。六角格网格的几何计算：格子中心在哪、像素点落在哪个格子上、两个格子隔多远、相邻格子有哪些。用的是标准算法（RedBlobGames）。

**地形效果**。每种地形（平原、村庄、城镇、河流、核心）有自己的移动消耗和防御加成，写在映射表里。

坐标算法通用，地形效果是游戏数据——后者应该配置化，目前还硬编码在代码里。

---

## 战斗（Combat）

纯计算函数。输入攻方、守方、战役状态、地形，输出命中/伤害/战报文本。

不改任何传入的对象。调用方拿到结果后自己决定怎么应用（扣血、标记、追加日志）。

规则概括：命中率从 55% 起步，受攻击力、士气和地形防御影响后取 5%-92%。命中后默认伤害 1，暴击（25% 概率）伤害 2，突击能力对掩体目标直接 2 点。

---

## 战役状态（Campaign）

战斗中变化的值，供战斗和界面查询：

- **战力**（0-100+）：影响命中率和敌方移动力。随每次行动衰减。
- **工事强度**：攻占核心需要降到的值。被攻克核心目标时减 1。
- **增援标记**：各支援军是否已到达（Map，key 是增援标识）。
- **胜负标记**：是否结束、谁赢了、详情文案。

附带了几个计算值：战力描述（充沛/尚可/吃紧/极限）、命中修正、敌方移动修正。

---

## 引擎外观（Game）

不存状态，把上述模型串成有用方法：

- **移动范围**：BFS 从当前格子开始走，扣地形消耗，到移动力耗尽为止。挡路的己方单位算障碍，核心阵地只有突击兵种能进。
- **攻击目标**：敌方已暴露单位中射程内的那些。隔一格以上打不中掩体内的目标。
- **增援**：到第几回合了？这个增援到过没？都没问题就创建单位追加日志。
- **胜负判定**：敌军全灭或核心被攻占 → 胜利。
- **单位查询**：哪个格子谁站着。

---

## 战役配置（CampaignConfig）

战役配置负责把 JSON 文件变成上面这些模型能用的数据：

- 读 `campaign.json` → 战役参数 + 增援波次
- 读 `map.json` → 地形网格
- 读 `units.json` → 兵种模板 + 初始单位部署

配置在开局加载一次，之后不再读。框架战役逻辑（胜负条件、地形效果表）目前还在代码里写死，不在 JSON 中。
