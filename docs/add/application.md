# 应用层

应用层接事件、调规则、改状态、通知 UI。一个循环体。

## 回合流程

```
玩家阶段
  ├── 选单位 → 显示可移动/攻击范围
  ├── 点空地 → 判断是否在范围内 → 移动
  ├── 点敌方 → 判断是否在范围内 → 攻击结算 → 扣血/标记行动
  ├── 点己方另一单位 → 切换选中
  ├── 长按 → 取消选中
  └── 结束回合 → 进入 AI 阶段

AI 阶段
  ├── 检查增援（第几回合了？这个增援到过没？）→ 生成单位
  ├── 每个已暴露的敌方单位 → 找射程内目标 → 攻击
  └── 回合数 +1 → 回到玩家阶段

任意时刻满足胜利条件 → 游戏结束
```

## 增援 ID

增援不预先分配 ID，生成时取当前最大 ID +1，避免冲突。

## 状态与 UI 的同步

状态改完后通知监听方，UI 收到通知重新读取需要的数据渲染。状态持有者不直接操作 UI。
