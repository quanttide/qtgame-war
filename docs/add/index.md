# 架构设计文档

## 分层

```
表现层  →  应用层  →  领域模型
                      ↑
                   数据层（配置）
```

箭头是依赖方向。下层不知道上层存在。

## 子文档

| 文档 | 讲什么 |
|------|--------|
| [domain.md](domain.md) | 规则和模型：兵种、地形、战斗、战役、引擎。以及配置和状态的区别。 |
| [application.md](application.md) | 流程控制：回合怎么转、增援怎么生、状态怎么同步。 |
| [presentation.md](presentation.md) | 界面和输入：地图怎么画、点击怎么处理、面板怎么分。 |
| [evolution.md](evolution.md) | 可扩展性：加新战役要改什么、哪些已经配置化、哪些还硬编码。 |
