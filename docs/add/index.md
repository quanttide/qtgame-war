## 核心理念

本项目本质是**叙事引擎 + 推演引擎**的复合体，覆盖两种玩法模式：

| | 战役层（互动剧本） | 战术层（兵棋推演） |
|---|---|---|
| 玩家角色 | 指挥官（下命令、等报告） | 指挥官（部署部队、下达战术指令） |
| 操作单元 | 决策选项（A/B/C/自定义） | 六角格单位 |
| 核心约束 | 信息不对称 + 指挥官立场 | 移动力 / 攻击范围 / 地形 |
| 反馈形式 | 战报文本 + 态势变化 | 地图变化 + 战斗结算 |
| 时间模型 | 事件脉冲（按战役速度推进） | 回合制（玩家→AI循环） |

## 架构分层

整个系统分为四个严格分离的层次：

```
┌──────────────────────────────────────────────┐
│              表现层 (Presentation)             │
│  - 地图渲染（六角格 SVG/Canvas）               │
│  - UI 组件（信息面板、日志、决策选项）           │
│  - 叙事生成器（将状态翻译为情境文本）            │
│  - 输入适配器（鼠标/触摸 → 游戏事件）           │
├──────────────────────────────────────────────┤
│            应用层 (Application)                 │
│  - GameController (ChangeNotifier)             │
│  - 事件分发与协作                              │
│  - 副作用管理（AI 延迟、动画、时间脉冲）         │
├──────────────────────────────────────────────┤
│              领域层 (Domain)                    │
│  - 六角格坐标系统 & 寻路                       │
│  - 战斗结算器                                  │
│  - AI 决策策略                                 │
│  - 指挥官意图系统 & 选项过滤器                  │
│  - 情报不对称模型（可信度 / 延迟 / 来源）       │
│  - 部队红线校验（体力 / 士气 / 编制 / 韧性）    │
│  - 规则引擎（可配置的胜利条件 / 后果计算）       │
├──────────────────────────────────────────────┤
│           基础设施层 (Infrastructure)            │
│  - 审计日志 / 事件溯源（记录、回放、分支推演）    │
│  - 持久化（本地存储 / 服务端同步）               │
│  - AI 适配器                                    │
└───────────────────────────────────────────────┘
```

核心原则：
- 领域层：纯 Dart 代码，不依赖 Flutter
- 应用层：GameController 驻留于此（ChangeNotifier），负责接收事件、协调领域对象、更新状态
- 表现层：纯 UI 渲染，通过 ListenableBuilder 订阅状态变化，不含业务逻辑
- 基础设施层：日志持久化、AI 适配、存储等外部关注点
