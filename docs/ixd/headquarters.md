# 指挥部看板

收到。三栏布局保留，它是骨架。我把“桌子”安进去。

—

指挥部看板 · 原型重构

一张桌子，三个区域

```
左区：情报桌面          中区：作战地图          右区：命令桌面
（摊开的文件堆）        （铺满的地图）          （起草中的命令稿）
```

不是左中右的UI面板，是桌面上三个物理分区。卫星图、纸文件、铅笔散落其间。

—

左区：情报桌面

桌上摊着的是什么？不是卡片列表，是情报原件：

· 电台监听记录纸（字迹潦草，有涂抹）
· 侦察兵带回的便条（皱，有水渍）
· 缴获的敌军文件（折痕，繁体竖排）
· 俘虏口供笔录（按手印）
· 地方送来的情报（宣纸，毛笔字）

玩家动作：翻动、对照、勾画。在纸上直接圈出可疑处、旁批存疑。看完推给一边，或压在采信区。

情报科长不弹出对话框，他站在左区桌边，当玩家翻到某份文件时开口：“这份骑侦报告和技侦截获对不上。”

—

中区：作战地图

铺满桌面，压着四角。上面有：

· 红蓝铅笔画的箭头、圈注（擦了又画的痕迹）
· 木头棋子代表团以上单位（可以拿起来、挪位置）
· 透明纸叠层（情报科标敌军可能位置）
· 标尺、铅笔、橡皮散落旁边

玩家动作：挪棋子、画箭头、拉测距线、开关叠层。一切直接在地图上操作，不弹窗。

签署命令后，棋子被作战科长按住：“不能动了。”

—

右区：命令桌面

摊开的作战日志、兵力表、预案对比、命令草稿。

命令稿不是表单，是一张纸。格子纸，钢笔字，有修改杠和旁注。玩家在纸上直接改——“拂晓前到达”改成“凌晨2时前到达”。改完，推到签署区。

签署不是点按钮，是把笔拿起来，在命令稿签名栏写下名字。落笔即生效，不可撤销。

—

时间：桌子中央的怀表

怀表开着盖，指针在走。总攻倒计时刻在表壳内侧。一秒一秒，全场同步。

不操作的时候，怀表的滴答声就是界面。

—

状态：只有两个

签署前：可以动棋子、改命令、换预案。主动思考状态。
签署后：所有东西按住不能动。只能看、等、接电报。承担后果状态。

一条线跨过去，体验质变。
