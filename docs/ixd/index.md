# 交互设计文档

## 核心设计问题：六角格 vs 即时战略（RTS）

### 问题陈述

原型使用六角格地图 + 回合制决策。考虑是否放弃六角格，改为红警式即时战略玩法以增强代入感。

### 分析框架

以 docs/index.md 定义的设计意图为评判标准：

> 还原真实战场指挥官的视角。让玩家体验那种"明知道历史走向，却依然感到步步惊心、决策艰难"的压迫感。

### 对比

| 设计意图要素 | 六角格兵棋（当前） | 红警式 RTS（候选） |
|------------|------------------|------------------|
| 指挥官视角 | ✅ 下命令、等报告、看地图 | ❌ 直接操控每个单位是排长视角 |
| 情报模糊/筛选判断 | ✅ 信息分级（可信/存疑/证伪）内置 | ⚠️ 战争迷雾有，但筛选判断机制需另建 |
| 决策艰难、步步惊心 | ✅ 回合制给思考空间，每条路有代价 | ⚠️ 实时产生压迫感，但决策质量下降 |
| 明知道历史走向 | ✅ 回合间可回顾、反思 | ❌ 实时节奏不允许停下来想 |
| 视觉代入感 | ⚠️ 抽象色块+文字标签，偏"兵棋" | ✅ 俯视角+单位造型，直观 |

### 结论

**不推翻六角格，增强其视觉代入感。**

红警式 RTS 与"指挥官视角"的设计意图冲突：
- 指挥官不亲手点射坦克——那是前线军官的事
- 实时操作挤压了"决策艰难"需要的思考空间
- docs 的参考对象（蘇北寒士兵棋推演）本身就是回合制兵棋

用户感受到的"代入感不足"来自**视觉抽象程度**，而非底层网格结构。

### 视觉增强方向（不改变六角格）

| 当前 | 目标 | 改动量 |
|------|------|--------|
| 纯色填充格子 | 地形纹理（草地/沙地/雪地） | CSS 纹理或 SVG pattern |
| 单位用文字标签（"敌装甲"） | 单位用图标（小坦克/小步兵图标） | CSS 或 SVG 图标 |
| 颜色区分地型 | 纹理+颜色双重编码 | 兼容色盲 |
| 静态格子 | 选中/悬停微动效 | CSS transition |
| 地图无缩放 | 鼠标滚轮缩放 | JS 添加 |

### 保留六角格的理由

1. **回合制是"指挥官视角"的前提**——没有回合制，情报分级和决策反思都无法实现
2. **六角格是兵棋推演的核心载体**——方向、距离、地形遮挡都有精确数学表达
3. **改动成本低**：当前 hexData + SVG 渲染架构支持在不改核心逻辑的前提下替换视觉
4. **兼容 docs 设计意图**：docs 写的"六边形单元格"不是随意选的，它支撑了"符合真实战场比例"的建模要求

## 已实现的增强

基于上述分析，实际完成了一轮迭代：

| 增强 | 实现方式 | 状态 |
|------|---------|------|
| 地形纹理 | SVG pattern 覆盖（4 种纹理：点阵/竖纹/三角/波浪） | ✅ |
| 单位图标 | Unicode 符号（▲装甲 ◆炮兵 ●步兵）+ 小字标签 | ✅ |
| 悬停动效 | CSS `filter: brightness(1.3)` + `transition: 0.15s` | ✅ |
| 滚轮缩放 | JS wheel 事件，0.4x–2.0x | ✅ |
| **指挥官心态面板** | HUD 半透明条，显示压力/疲劳/信心 + 私人事务信件 | ✅ 新增 |

### 指挥官心态 HUD

将"代入感"从视觉延伸到心理层面。HUD 显示三项指标：

- **精神压力**：战场决策累积的心理负担，正面冲锋比侧翼包抄产生更多压力
- **疲劳度**：随时间上升，连续作战消耗指挥官精力
- **信心**：胜利/失败直接影响，防守反击失败的信心打击是进攻失败的两倍

每 3-4 回合触发私人事务（家信/通知），在决策间隙提醒玩家"你是一个有生活的人"。

心态面板作为 HUD 叠加在地图上方，确保玩家在做每个决策前都能看到自己指挥官的状态——**把"步步惊心"从口号变成可感知的数值**。
